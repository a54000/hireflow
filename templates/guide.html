HR Guru ATS User Guide

Use this guide for day-to-day recruiter workflows: creating clean requirements, adding candidates, uploading in bulk, sending daily reports, and searching the internal database.

Quick Start

  1. Start from the Dashboard action boxes.
  2. Create or select the correct requirement before adding candidates.
  3. Check candidate details after parsing, especially email, phone, skills, experience, notice period, and requirement mapping.
  4. Use Daily Reporting to share selected candidates for the current day or previous days.
  5. Use Power Search when you need to find candidates already in the database.

Add New Requirement

  1. From Dashboard, click Add Requirement.
  2. Select the correct client.
  3. Enter a clean requirement title using the naming standard.
  4. Fill Location, Experience, and Degree / Education Criteria. Location is mandatory.
  5. Add detailed JD text or attach the JD file. One of these is mandatory.
  6. Click Parse JD if you want the system to fill skills from the JD.
  7. Primary Skills is optional, but recommended when the JD parser cannot identify skills clearly.
  8. Click Save Requirement.

Requirement naming standard

RuleExample
Use role name only.Java Developer, Finance Analyst, Data Engineer
Do not include client name in the title.Use Client field for Taggd-Siemens; title should be Java Developer.
Do not include numbers.Use Java Developer, not Java Developer 2 or Java 5 Developer.
Do not use generic words.Avoid JD, Requirement, Urgent, Opening, Hiring.
Keep spelling and punctuation clean.Use common separators only, such as -, /, &, +, #.

Add Single Candidate

  1. From Dashboard, click Add Candidate.
  2. Upload the candidate CV and parse it.
  3. Select the requirement before saving the candidate.
  4. Review parsed details: candidate name, email, phone, current company, designation, total experience, current location, preferred location, notice period, salary, and skills.
  5. Correct any missing or wrongly parsed information.
  6. Save the candidate.

Things to check before saving

  • Email and phone number are correct.
  • Requirement and client mapping are correct.
  • Primary skills match the requirement.
  • Experience and notice period are realistic.
  • CV is attached and visible.

Bulk Candidate Upload

Excel + CV upload

  1. From Dashboard, click Bulk Candidate Upload.
  2. Create the Requirement first from Add Requirement, including the mandatory Taggd Recruiter Name.
  3. Download or use the approved Excel template.
  4. Fill candidate details carefully in the template.
  5. Ensure Requirement title and Client in the spreadsheet match an existing Requirement.
  6. Keep CV files ready and ensure filenames match the template references where required.
  7. Upload the Excel file and CVs.
  8. Review the upload result for skipped rows, duplicates, or missing information.
Bulk upload does not create Requirements. If a Requirement is missing, that row is skipped with a Requirement not found reason. Taggd Recruiter is selected on the Requirement, not in the candidate spreadsheet.

Google Sheet import

Use Google Sheet import when a team maintains candidate data in shared sheets. The ATS reads the selected tabs, maps sheet headings to ATS candidate fields, and imports candidates without CV files. Position Title must match an existing Requirement.

  1. From Dashboard, click Bulk Candidate Upload.
  2. Create each Requirement first from Add Requirement, including Taggd Recruiter Name.
  3. Select Google Sheet Import.
  4. Paste the Google Sheet URL.
  5. Enter the tabs to import, such as May, April. Leave blank only when you intentionally want all tabs.
  6. Enter Default Client if the sheet does not have a Client column.
  7. Keep Only read new rows since last import checked for regular imports.
  8. Click Preview.
  9. Review Column Mapping. Change any incorrect mapping before import.
  10. After changing a mapping, click Preview again.
  11. Click Import Ready Candidates.
Sheet headingATS mapping / behavior
Position TitleUsed to find an existing Requirement. If the Requirement does not exist, the row is skipped until the Requirement is created.
Client or Client NameUsed for requirement/client mapping.
StatusImported as candidate status.
Added DateImported as candidate added date.
Industry/DomainImported into candidate Industry / Domain.
Sourcer/Recruiter NameImported as Sourcer / Recruiter Name.

Smart new-row import

  • The first import reads the selected sheet tabs normally.
  • After each successful batch, ATS remembers the last processed row number for each tab.
  • Next time, ATS reads only rows added after the last processed row, which is faster for large sheets.
  • If the import stops midway, click Import Ready Candidates again. Already imported rows from that batch are skipped.
  • Turn off Only read new rows since last import only when you need a full re-scan, such as when old rows were edited or rows were inserted in the middle.
Best practice: add new records at the bottom of the Google Sheet. Avoid inserting candidate rows above already imported rows unless you plan to run a full re-scan.

Daily Reporting

  1. From Dashboard, click Daily Reporting.
  2. Click Export to download selected candidates locally.
  3. The export downloads a zipped file containing the Excel report and the respective CVs.
  4. Click Send Report to email selected candidates to the Taggd team.

Daily reporting checks

  • Only selected candidates are exported or emailed.
  • Use the correct date when sending reports for previous days.
  • Confirm candidate status and requirement before sending.
  • Check that CV links/files are available for selected candidates.

Power Search

  1. Open Power Search from Dashboard or the left menu.
  2. Enter job title, skills/keywords, years of experience, and location.
  3. Use focused keywords, for example Java Spring SQL or Finance SOX Audit.
  4. Run the search and review matching candidates from the internal database.
  5. Open candidate details before shortlisting or contacting.
Power Search works best when you combine role, skills, experience, and location. Avoid very broad searches unless you want a large result set.
