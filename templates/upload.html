Import Candidates

Create Requirements first, including Taggd Recruiter Name, then upload JD files, the completed Excel/CSV file, and matching CV files. Bulk upload maps candidates only to existing Requirements.

Before upload, create the Requirement from Add Requirement and select the Taggd Recruiter Name there. Bulk upload does not create Requirements. Required columns: Requirement, Candidate Name, Email, Phone, Current Company, Current Role, Experience, Key Skills, Notice Period, Current Salary, Expected Salary, Current Location, Preferred Location. Requirement can contain the existing Requirement title, ATS Requirement ID, or Requisition ID; CV files should match each row by email, phone, or candidate name. Rows with missing Requirements are skipped with a Requirement not found reason.
Import From Google Sheet

Paste a team Google Sheet URL, choose tabs, preview mapped candidates, then import without CV files. Position Title must map to an existing Requirement.

Reconnect Google Permissions
Use a Google Sheet URL. Position Title is used to find an existing Requirement; ATS no longer creates Requirements during import. Create the Requirement first with Taggd Recruiter Name. Smart new-row mode reads only rows added after the previous successful import.
